Docker es una herramienta de contenedorización que permite crear y ejecutar aplicaciones de manera aislada y portable en diferentes entornos. En el contexto de Santo Pegasus, se utiliza Docker para levantar el ambiente local con todas las dependencias necesarias, como la base de datos y Redis, y para asegurar la inmutabilidad del artefacto en el proceso de Integración y Entrega Continuas (CI/CD). Según la documentación, Docker es obligatorio para asegurar que el mismo código validado en Staging (homologación) sea promovido a Producción sin variaciones. Esto se logra a través de la contenedorización, que garantiza que el código se ejecute de manera consistente en diferentes entornos. En resumen, Docker es una herramienta fundamental en el desarrollo y despliegue de aplicaciones en Santo Pegasus, ya que permite crear entornos de desarrollo y producción consistentes y escalables, y asegura la integridad del código a lo largo del proceso de entrega.
